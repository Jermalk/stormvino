# SVP — Svelte Web Panel

> Internal name: SVP. Lives at `/opt/ov_server/monitor/`.
> Served by ov_server at `/monitor` when `monitor/dist/` exists.
> All data sourced from ov_server REST endpoints — no separate backend.

---

## Architecture

```
ov_server (port 11435)
  /health                  → ServerPanel
  /monitor/api/system      → GpuPanel + SystemPanel  (sysfs + psutil)
  /admin/profile-models    → ProfilerPanel (GET status, POST trigger)
  /admin/profile           → ProfilesPanel (POST switch)
  /admin/scope             → ProfilesPanel (POST)
  /maintenance/restart     → ProfilesPanel (POST)
  /monitor/api/metrics     → Charts (uPlot time-series, stub → Postgres)
  /monitor/api/model-usage → future model breakdown
  /v1/models               → future catalogue view

  GET /monitor             → serves monitor/dist/ (StaticFiles)
```

Build: `cd monitor && npm run build` → `dist/` (gitignored; rebuild on change)

## Roadmap

### Phase 2 — Live charts from Postgres ✓ COMPLETE

**Step 2.1 — `/monitor/api/metrics` SQL**
- `SELECT EXTRACT(EPOCH FROM completed_at)::int, tok_per_sec FROM request_log WHERE completed_at > NOW() - INTERVAL '<minutes> minutes' ORDER BY completed_at`
- Add `elapsed_sec`, `tokens`, `ttfb_ms` as selectable metrics
- Returns `{ts: [int...], values: [float...], metric, minutes}`

**Step 2.2 — Charts.svelte real data**
- Wire the uPlot chart to the real Postgres response
- Add metric selector tabs: tok/s · elapsed · tokens · TTFB
- Refresh every 30s; re-query on tab switch

**Step 2.3 — Model usage panel**
- `/monitor/api/model-usage` SQL: `SELECT model_id, COUNT(*), AVG(tok_per_sec), SUM(tokens) FROM request_log GROUP BY model_id`
- New `ModelUsage.svelte` panel with a bar chart or ranked table

## Dev workflow

```bash
# Watch mode (local dev, port 5173)
cd /opt/ov_server/monitor && npm run dev

# Production build + immediate server reload
cd /opt/ov_server/monitor && npm run build && sudo systemctl restart ov-server
```

The dev server proxies to ov_server via `vite.config.js` `server.proxy`.
Check `vite.config.js` for the proxy target — should be `http://localhost:11435`.
